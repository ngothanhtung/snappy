This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Snappy is a macOS menu-bar agent app (no Dock icon, `LSUIElement`) that snaps the
focused window to halves/thirds and moves it between displays via global keyboard
shortcuts. Built with Swift Package Manager (not an Xcode project), targeting
macOS 14+.

## Commands

```bash
swift test                          # run all SnappyCore unit tests
swift test --filter LayoutCalculatorTests          # one test class
swift test --filter LayoutCalculatorTests/leftHalf # one test method
swift build                         # debug build (compile check)
./Scripts/bundle.sh                 # release build + assemble Snappy.app
open ./Snappy.app                   # launch the assembled app
```

Tests only cover `SnappyCore` — the `Snappy` app target depends on AppKit /
Accessibility APIs and is not unit tested. After changing core logic, `swift test`
is the fast feedback loop; to verify actual window behavior you must `bundle.sh`
and launch (see Accessibility note below).

## Architecture

Two SPM targets, split by testability:

- **`SnappyCore`** — pure, dependency-free, fully unit-tested geometry. No AppKit,
  no side effects. Everything here is `static` functions on enums:
  - `LayoutCalculator.frame(for:in:)` — the layout math (halves/thirds → `CGRect`)
  - `Geometry.flipY` — converts between coordinate systems (see gotcha below)
  - `DisplayMapper` — maps a window proportionally onto another display
  - `DisplayOrdering` — picks prev/next display by screen position
- **`Snappy`** — the app layer (AppKit + Accessibility + KeyboardShortcuts dep).
  Keep this thin; push any non-trivial geometry down into `SnappyCore` so it can
  be tested.

Control flow: `Main` (sets `.accessory` policy) → `AppDelegate` installs
`StatusMenuController`, calls `HotkeyManager.registerAll()`, and requests
Accessibility. A shortcut or menu item produces an `AppAction`, which
`WindowManager.perform` dispatches: it builds a `Context` (focused window +
its screen via `WindowEngine`/`ScreenProvider`), computes the target frame with
`SnappyCore`, and writes it back through `WindowEngine.setFrame`.

### Single source of truth for actions

`ShortcutNames.swift` is the one place defining every action. `AppAction`,
`ActionBinding`, and `Bindings.all` drive hotkey registration
(`HotkeyManager`), the status menu (`StatusMenuController`), and the settings UI
(`SettingsView`). To add or change a shortcut/layout, edit this file (and add the
`WindowLayout` case + calculator branch in `SnappyCore` if it's a new layout) —
the rest wires up automatically.

### Coordinate systems (the central subtlety)

AppKit uses a global origin at the **bottom-left** of the primary screen (y up);
the Accessibility API uses **top-left** (y down). `WindowManager` reads frames in
AX coords, flips with `Geometry.flipY(_:primaryHeight:)` where `primaryHeight` is
`NSScreen.screens.first?.frame.height`, does all layout math in AX/top-left space,
then writes AX coords back. The flip is its own symmetric transform. When touching
window placement, be explicit about which space a `CGRect` is in.

`WindowEngine.setFrame` sets position → size → position again on purpose: some apps
clamp the new position against the old size on the first pass.

## Accessibility permission + code signing (critical for testing the real app)

macOS ties the Accessibility (TCC) grant to the app's **code signature**.
`bundle.sh` defaults to ad-hoc signing (`codesign --sign -`), which produces a new
signature every rebuild, **silently invalidating** the previous grant — window
snapping then fails quietly. Two ways to deal with it:

- Quick: after each ad-hoc rebuild, remove and re-add Snappy in **System Settings
  → Privacy & Security → Accessibility**.
- Stable: create a self-signed Code Signing cert in Keychain Access, then
  `export SNAPPY_SIGN_IDENTITY="Snappy Self-Signed"` before `bundle.sh`. The grant
  survives rebuilds. (See README.md for full steps.)

`WindowManager.perform` and `AppDelegate` go through `AccessibilityPermission`,
which surfaces guidance instead of failing silently when untrusted.
